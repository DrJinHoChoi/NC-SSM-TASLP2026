## 3-1. 협업과제 소개

### 제안 배경

 ◦ 현재 스마트 가전의 음성 인식은 대부분 클라우드 서버에 의존하여 응답 지연(200ms~1s), 네트워크 의존성, 개인정보 유출 위험, 지속적 통신 비용이 발생함
   - 세탁기/건조기와 같은 생활가전은 네트워크 불안정 환경(지하실, 세탁실)에서 사용되는 경우가 많아 클라우드 기반 음성 인식의 한계가 두드러짐
   - 소음 환경(세탁기 탈수 진동 70~80dB, 건조기 작동음 60~70dB)에서 기존 온디바이스 경량 모델의 인식률이 급격히 하락(96% → 28% @-15dB)

 ◦ 본 과제는 자체 개발한 NC-SSM(Noise-Conditioned State Space Model) 기술을 기반으로, 4.8KB 초경량 모델로 소음 환경에서도 95%+ 인식률을 달성하는 완전 온디바이스 음성 인식 및 가전 제어 통합 솔루션을 개발함
   - 하위1(NLU 가전 제어)과 하위2(초경량 임베디드 최적화)를 단일 파이프라인으로 통합하여 웨이크워드 감지부터 가전 제어 명령 실행까지 엔드투엔드로 처리
   - 기존 AI Glass 프로젝트에서 검증된 KWS→STT→NLU→제어 파이프라인을 LG 가전 환경에 맞춤 최적화

 ◦ 제품(솔루션) 개요 — NC-SSM + NC-TCN 듀얼 모델 전략
   - **NC-SSM** 온디바이스 음성 인식 엔진: 7,443 파라미터, INT8 4.8KB, Cortex-M7에서 0.94ms 추론 — 극한 경량화 시나리오(코인셀 전원, sub-10KB 플래시)
   - **NC-TCN** 온디바이스 음성 인식 엔진: 21,689 파라미터, Cortex-M7에서 5.4ms 추론, 96.6% 정확도 — 고정확도 시나리오(MCU 메인 전원)
   - **공유 NC Frontend**: 두 모델 모두 동일한 SNR Estimator + Learned Spectral Gate(LSG) + DualPCEN-v2 노이즈 적응 프론트엔드 공유 → 노이즈 강건성의 핵심은 프론트엔드에 있으며, 백엔드(SSM/TCN)는 배포 환경에 맞게 교체 가능
   - 12-Class KWS 기반 가전 제어 인터페이스: 10개 음성 키워드(on/off/go/stop/up/down/left/right/yes/no)로 세탁기/건조기 전체 기능 제어
   - 한국어 문맥 인식 NLU 엔진(확장): 사용자 발화 의도 분석 → 기기/동작/파라미터 추출 (복합 명령 지원)
   - 가전 제어 미들웨어: KWS/NLU 결과를 LG ThinQ/가전 제어 API로 변환
   - 통합 개발 SDK: C SDK + Python SDK, ONNX/TFLite 내보내기 지원

### 12-Class KWS 기반 가전 제어 매핑

NC-SSM이 인식하는 12개 키워드(Google Speech Commands V2)를 가전 제어 명령으로 직접 매핑하여, 별도의 NLU 없이도 MCU 단독으로 가전 제어가 가능함:

| 키워드 | 기능 | 동작 예시 |
|--------|------|-----------|
| LEFT / RIGHT | 기기 전환 | 세탁기 → 건조기 → 에어컨 → 조명 → TV 순환 |
| ON / OFF | 전원 제어 | 선택된 기기 전원 켜기/끄기 |
| GO | 동작 시작/재개 | 세탁 시작, 건조 시작 |
| STOP | 일시정지 | 동작 중 정지 |
| UP / DOWN | 파라미터 조정 | 세탁기: 온도 ±10°C / 에어컨: ±1°C / 조명: 밝기 ±20% / TV: 채널 ±1 |
| YES | 모드 순환 | 표준→울→이불→급속→헹굼→탈수→에코 순환 |
| NO | 설정 초기화 | 모든 파라미터를 기본값으로 리셋 |

→ 이 매핑 방식을 통해 KWS 모델(7,443 params, 4.8KB) 하나만으로 세탁기/건조기의 전원, 모드, 온도, 동작을 모두 제어 가능. MCU 단독 동작으로 추가 NLU 모듈이나 클라우드가 전혀 필요 없음.

### NC-SSM vs NC-TCN 모델 비교

| 항목 | NC-SSM (극한 경량) | NC-TCN (고정확도) | 비고 |
|------|-------------------|------------------|------|
| 파라미터 | **7,443** | **21,689** | NC-TCN은 DS-CNN-S(23.8K)보다 적음 |
| 추론 지연시간 | 0.94ms | 5.4ms | 둘 다 Cortex-M7 480MHz |
| MACs | 0.68M | 2.1M | NC-TCN은 DS-CNN-S 대비 10.9x 적음 |
| SRAM | 23.8KB | 85KB | |
| 클린 정확도 | 95.1% | **96.6%** | NC-TCN이 1.5%p 높음 |
| 0dB 노이즈 | 89.6% | **89.7%** | 동등 수준 |
| -15dB 노이즈 | 71.3% | 72.8% (+SS) | NC-TCN+SS가 우위 |
| 배터리(CR2032) | **288일** | ~45일 | NC-SSM이 극한 저전력 |
| 병렬화 | 순차 스캔 | **완전 병렬** | TCN은 SIMD 최적화 |
| INT8 양자화 | 제한적 | **우수** | TCN이 양자화 친화적 |
| 배포 시나리오 | 코인셀, sub-10KB | 메인전원 MCU | 타겟에 따라 선택 |

→ **핵심 발견**: 노이즈 강건성의 주요 원천은 공유 NC Frontend(SNR Estimator + LSG + DualPCEN)이며, 백엔드(SSM/TCN)는 배포 환경에 맞게 교체 가능. LG 세탁기/건조기에는 메인전원 MCU 사용이 가능하므로 **NC-TCN-20K가 1차 타겟**, NC-SSM은 배터리 구동 리모컨/부속기기용.

| 항목 | 사양 |

## 3-2. 협업과제 기술 경쟁력

 ◦ 핵심 기술 혁신성: SSM 노이즈 전파 문제의 세계 최초 수학적 해결
   - 기존 선택적 SSM(Mamba)은 입력 의존 파라미터(Δ, B, C)가 노이즈와 곱셈적으로 결합하여 출력 분산이 O(σ_n^6)으로 폭증하는 근본적 문제 존재
   - NC-SSM은 per-sub-band 선택성 게이팅으로 40개 멜 밴드를 6개 서브밴드로 그룹화하여 주파수별 독립 노이즈 적응을 수행, 출력 분산을 CNN급 O(σ_n^2)로 제어
   - 고 SNR 구간에서는 빠른 시간 적응(음성 전달), 저 SNR 구간에서는 긴 메모리(노이즈 억제)로 자동 전환

 ◦ 경쟁 기술 대비 차별성
   - vs Qualcomm BC-ResNet-1 (7,464 params): NC-SSM이 동일 파라미터 수에서 연산량 6.8배 절감, 배터리 수명 9.3배 연장
   - vs Google DS-CNN-S (23,756 params): NC-TCN이 유사 파라미터(21.7K)에서 MACs 10.9배 절감, 지연시간 2.1배 단축, 정확도 동등(96.6%)
   - vs Picovoice/Sensory 상용 솔루션: 10~100배 더 작은 모델 크기로 동등 노이즈 강건성
   - 핵심 차별점: NC Frontend(SNR Estimator + LSG + DualPCEN)가 모델 내부에서 노이즈 적응 수행 → 별도 전처리(GTCRN 등) 불필요, 추가 파라미터/지연 없음
   - 듀얼 백엔드 전략: 동일 NC Frontend에 SSM(극한 경량) 또는 TCN(고정확도) 백엔드를 교체 장착 → LG 가전 제품별 최적 모델 선택 가능

 ◦ 기술 완성 수준 (TRL 4~5)
   - 학술 검증: Interspeech 2026 국제학술대회 논문 투고 완료
   - 지적재산권: 한국 특허 출원 완료 (NC-SSM + DualPCEN + MoE 라우팅), 미국 특허 출원 준비 중
   - 소프트웨어: C SDK (MCU 배포), Python SDK (학습/추론), ONNX/TFLite 내보내기, 스트리밍 추론 엔진 구현 완료
   - 하드웨어 검증: ARM Cortex-M4/M7/M33/M55에서 추론 벤치마크 완료, FPGA 시뮬레이션 수행
   - 실시간 데모: CES 2027 라이브 데모 서버, AI Glass 멀티모달 데모 시스템 운영 경험

   - 스마트 가전 음성 제어 시장: 연 30%+ 성장, 2030년 $8B 규모 예상
   - 온디바이스 전환 가속: 프라이버시 규제 강화(GDPR, 개인정보보호법) → 클라우드 의존 탈피 수요 급증

 ◦ LG전자에 미치는 영향
   - 세탁기/건조기 프리미엄 라인업 차별화: 클라우드 없이도 완전한 음성 제어 → 네트워크 미연결 가전에도 AI 탑재 가능

## 6. GPU 활용계획

### 6-1. GPU 활용 필수성

 ◦ NC-SSM 모델 학습 및 한국어 웨이크워드 파인튜닝에 GPU 연산이 필수
   - 한국어 음성 명령 데이터셋 학습: 8만+ 샘플 × 30 에폭 × 40+ 모델 변형 실험
   - 노이즈 강건성 실험: 5종 노이즈 × 7개 SNR 레벨 × 모델 변형 = 수백 회 학습 사이클

 ◦ 온디바이스 배포 모델은 MCU에서 동작하지만, 학습 파이프라인은 GPU 기반
   - 양자화 인식 학습(QAT): INT8 정밀도에서의 정확도 최적화에 GPU 필수
   - NLU 모델 학습: 의도 분류기 및 슬롯 태거 학습

### 6-2. GPU 활용 타당성

 ◦ PyTorch 기반 학습 파이프라인 기 구축 완료 (train_all_models.py)
   - AdamW 옵티마이저, 코사인 어닐링 스케줄러, 라벨 스무딩 적용
   - 데이터 증강: 시간 이동(±100ms), 볼륨(±20%), 가우시안 노이즈(p=0.3)

 ◦ GPU 사용 규모: A100 40GB × 2장, 협약기간 내 약 2,000 GPU-hours 예상
   - 모델 학습 1회: ~30분 (배치 64, 30 에폭)
   - 전체 실험 (40+ 변형 × 노이즈 × 반복): ~1,500시간
   - NLU 모델 학습: ~500시간

### 6-3. GPU 활용 기대효과

 ◦ 성능 측면: GPU 활용으로 학습 시간 10배 단축 (CPU 대비), 더 많은 하이퍼파라미터 탐색으로 최적 모델 도출
 ◦ 비용 측면: 학습 완료 후 배포 모델은 $1 미만 MCU에서 동작 → 클라우드 추론 비용 제로
 ◦ 경쟁우위: 빠른 실험 사이클로 LG 가전 맞춤 최적화 모델을 단기간에 도출
